How to Think in React

What exactly is React?

  • A library for building user interfaces

  • Uses components to enforce separation of concerns

  • Makes you think in terms of components, state and props

  • Strong emphasis on declarative programming and immutability

Components, State and Props

  • Component:

    A component is a JavaScript function that takes props as input and returns React elements describing what should appear on screen

  • State:

    A component's state is a simple JavaScript object that contains data specific to the component

  • Props:

    The props of a component is a JavaScript object containing any data that the parent component has passed to the child component

Component Life Cycle

  • Component receives props and renders the React element using data from props

  • User interacts with the UI and changes the state of the app

  • If that state is changed it re-renders the entire component with updated state

Example App: Youtube Search App

Search Form

class SearchForm extends Component { constructor() { super(); this.handleChange = this.handleChange.bind(this); this.handleSubmit = this.handleSubmit.bind(this); this.state = { query: "" }; } static propTypes = { searchHandler: PropTypes.func }; handleChange(query) { this.setState({ query }); } handleSubmit(evt) { evt.preventDefault(); this.props.searchHandler(this.state.query); } render() { return (
this.handleChange(evt.target.value)} className="search-input" type="text" placeholder="Search" />
); } }
